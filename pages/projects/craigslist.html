Background & Overview


The goal of this project is to practice my SQL and Python skills using a real-world dataset. As a car enthusiast and someone who has spent a lot of time on Craigslist buying and selling bicycles, I designed this project around a Kaggle dataset I found that has automotive classified listings from Craigslist.

Conclusions


Through the ETL process, the dataset was reduced from over 400,000 listings to 227,475 (-46%). This removed some outliers, null values, and listings with messy inputs.

In the end, the dataset was loaded into a SQLite database. The database has 3 tables -- one with the 227 thousand cleaned listings, and the other two with the original number of listings. The listings table houses most of the relevant information we expect to use.

The location table includes all the location information for these listings, and the description table simply holds the description for each of the listings.


The primary key that links these three tables is the ID column.



In the next part of this project, I will be using this database to create a dashboard to visualize the data.
